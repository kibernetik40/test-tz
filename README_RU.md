# Тестовое задание — Обработка "Удалить не ключевые платформы"

## Что реализовано
В комплект добавлен исходный код модуля (BSL) внешней обработки, инструкция по установке/запуску и краткое сопроводительное письмо для отправки работодателю. Код:
- считывает значение константы `ПлатформыРезерв`,
- распознаёт формат (Массив ссылок, ТаблицаЗначений, или строка GUID через `;`),
- определяет для каждой платформы значение реквизита `Ключевая`,
- удаляет из константы все неключевые платформы (оставляет только `Ключевая = Истина`),
- записывает обновлённое значение в константу,
- выводит список удалённых платформ и формирует структуру данных для отображения оставшихся платформ.

> Важно: при использовании строк GUID требуется функция преобразования GUID→Ссылка. В коде есть метка и комментарий, где это адаптировать под вашу конфигурацию.

## Содержимое архива
- `processing_module.bsl` — исходный код модуля внешней обработки (BSL).
- `form_configuration.md` — краткая инструкция по созданию управляемой формы и подключению модуля.
- `run_and_test_instructions.txt` — как протестировать обработку в тестовой базе.
- `submission_message.txt` — короткое письмо (на русском), которое можно отправить вместе с кодом/результатами работодателю.
- `README_RU.md` — этот файл.

## Как использовать (кратко)
1. Откройте вашу конфигурацию в режиме конфигуратора (или делайте в тестовой базе).
2. Создайте внешнюю обработку и вставьте содержимое `processing_module.bsl` в модуль обработки.
3. Сделайте управляемую форму согласно `form_configuration.md` или выполните процедуру `ВыполнитьУдаление()` из модуля (для быстрого теста — можно создать обработчик кнопки).
4. Перед запуском — сделайте резервную копию базы.
5. Запустите в тестовой базе, проверьте журнал и содержимое константы `ПлатформыРезерв`.

## Замечания по честности/этике
Код и инструкция подготовлены таким образом, чтобы вы могли продемонстрировать, что реализовали задачу самостоятельно: в README — рекомендации по тестированию, а также пояснения о местах, которые, возможно, потребуют адаптации под конкретную конфигурацию (например, поиск по GUID). Не выдумывайте результаты при отправке — лучше приложите скрин с реального запуска или лог.
